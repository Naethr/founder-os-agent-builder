# SEO Research

## When to Use

Use this skill when Web Studio OS needs SEO topics, keyword hypotheses, page ideas, metadata drafts, or local SEO opportunities for a small service website.

## Expected Inputs

- Business type.
- Services offered.
- Target location or service area if provided.
- Target audience.
- Existing page or planned page.
- Validated competitor observations if available.
- Current offer and scope boundaries.

## Expected Output

- Keyword hypotheses grouped by intent.
- Page ideas and content angles.
- Local SEO notes.
- Draft title and meta description ideas.
- Internal linking suggestions when relevant.
- Validation reminders and uncertainty notes.

## Method

1. Identify the main service and customer intent.
2. Generate service, problem, comparison, and location-based keyword hypotheses.
3. Suggest pages that match distinct intents instead of forcing all topics onto one page.
4. Draft plain-language metadata ideas.
5. Add local SEO notes only when a real location or service area is provided.
6. State what must be validated before client delivery.

## Guardrails

- Do not claim real search volume unless a real SEO tool was used.
- Mark keyword ideas as hypotheses.
- Recommend validation before client delivery.
- Do not guarantee rankings, traffic, leads, or revenue.
- Do not scrape restricted sources or store competitor data without approval.

## Example

Input:
Generate SEO ideas for a local renovation services website.

Output:
Provide keyword hypotheses such as "kitchen renovation contractor [city]" and "bathroom renovation estimate [city]", page ideas for core services and project proof, local SEO notes about service area clarity, and a reminder that no search volume was validated.
