# Admin Sandbox

## When to Use

Use this skill to create fictional quotes, administrative summaries, scope checklists, payment reminder drafts, or safe commercial planning documents.

## Expected Inputs

- Requested service or website type.
- Scope assumptions.
- Page count or deliverables if known.
- Content readiness.
- Optional maintenance needs.
- Currency and fictional pricing model if provided.
- Admin rules and approval boundaries.

## Expected Output

- Fictional quote or admin draft.
- Line items or checklist items.
- Assumptions used.
- Missing information.
- Approval requirement.
- Clear statement that the output is not final or legally/accounting valid.

## Method

1. Confirm whether the request is for internal planning or client-facing use.
2. Use only provided or documented assumptions.
3. Separate setup work from maintenance work.
4. Flag missing billing, scope, tax, or legal details.
5. Mark every quote as fictional until human validation.
6. Provide the safest next admin action.

## Guardrails

- Quotes are fictional drafts only.
- No accounting, legal, tax, or compliance claim.
- Human validation is required before client use.
- Paid services, taxes, invoices, and official pricing must be checked by a human.
- Do not record real accounting data or store sensitive billing data without approval.

## Example

Input:
Create a quote draft for a small renovation showcase website with five pages and optional maintenance.

Output:
Produce a fictional internal quote draft with setup and maintenance sections, assumptions about pages and content readiness, missing billing details, and an explicit warning that it cannot be sent or used commercially before human review.
