# Learning Coach

## When to Use

Use this skill when the founder needs to decide what to learn next based on project goals, vault notes, execution gaps, and current Web Studio OS priorities.

## Expected Inputs

- Founder question.
- Relevant vault notes when available.
- Current project goal.
- Known blockers or skill gaps.
- Recent run evidence or decisions.

## Expected Output

- Source-supported observations.
- Practical learning priorities.
- Short learning plan.
- Exercises or deliverables for the week.
- Risks of learning the wrong thing.
- Source citations when vault notes are used.

## Method

1. Read relevant notes when available.
2. Cite exact note paths used.
3. Separate facts from recommendations.
4. Identify the highest-leverage learning gap for the current project stage.
5. Recommend a short plan with concrete outputs.
6. Keep the plan realistic and directly tied to Web Studio OS.

## Guardrails

- Use notes when available.
- Cite sources when using vault notes.
- Avoid vague motivation.
- Produce a realistic short learning plan.
- Do not invent market facts, client needs, or validated results.
- Do not write permanent notes without approval.

## Example

Input:
What should I learn this week to better sell Web Studio OS?

Output:
Use vault notes about the offer, sales tone, target clients, and learning priorities. Recommend focused learning on offer clarity, discovery questions, quote framing, and one sales email practice exercise, with citations to the notes used.
