# Run Journal

## Day 1 Entry

Founder OS Blueprint was chosen to create a clear foundation before building tools. The risk at this stage is not technical difficulty. The bigger risk is building an over-complicated agent system before the business workflow is understood.

The selected default case is Web Studio OS because it is concrete, small enough for an MVP, and has repeatable founder tasks: prospect research, offer analysis, sales drafting, SEO preparation, product scoping, admin drafts, and delivery review.

## Most Important Agents

The most important early agents are:

- **Orchestrator:** because uncontrolled agent autonomy would create messy workflows and unsafe actions.
- **Mail / Sales Agent:** because sales communication is frequent and sensitive.
- **Code / Product Agent:** because client work must translate into clear scopes and checklists.
- **Admin / Accounting Agent:** because quotes, invoices, and scope assumptions need careful validation.

The SEO and Prospecting Agents are useful, but they should remain tightly scoped. They can create leverage only if the system avoids spam, unsupported claims, and careless data storage.

## Sensitive Actions

The most sensitive actions are:

- Sending real emails.
- Contacting leads.
- Importing or storing prospect data.
- Generating or sending quotes.
- Changing pricing.
- Publishing public content.
- Updating client deliverables.
- Using paid APIs.
- Pushing to a remote repository.
- Recording real accounting data.

These actions require human validation because they create external consequences, financial implications, privacy obligations, or project history.
