# Learning Plan

## Purpose

Guide the founder toward the most useful learning for launching Web Studio OS without overbuilding.

## Learning Priorities

- Refine the launch offer until it can be explained simply and scoped quickly.
- Learn discovery questions that expose service, audience, proof, content readiness, and exclusions.
- Learn basic quote framing: setup scope, maintenance scope, assumptions, and review gates.
- Learn landing page conversion basics: headline, service promise, proof, CTA, objections, and form.
- Learn local SEO basics that support service pages without making guarantees.
- Practice one short sales email draft that asks for missing scope before pricing.
- Learn permission boundaries so agents do not send, publish, quote, or store sensitive data without approval.

## Suggested Weekly Rhythm

- Spend the most time on offer clarity and discovery.
- Practice with one fictional but realistic service business.
- Keep notes short and update them only after reviewing the result.
- Measure progress by whether the founder can explain, scope, and draft the offer more clearly.
