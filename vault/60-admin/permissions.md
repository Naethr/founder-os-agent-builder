# Permissions

## Purpose

Define what agents may and may not do when using Web Studio OS memory.

## Allowed Without External Action

- Read local vault notes.
- Produce internal drafts with citations to source notes.
- Summarize documented decisions.
- Recommend next learning steps when clearly marked as recommendations.
- Create evidence files when the run actually happened.

## Requires Human Validation

- Sending emails or proposals.
- Publishing pages or public content.
- Quoting prices, discounts, or delivery timelines.
- Storing personal, client, billing, or prospect data.
- Updating official business decisions.
- Using paid APIs, external platforms, or cloud knowledge bases.
- Pushing to a remote repository.

## Agent Boundaries

- Agents must not invent facts that are not in the vault.
- Agents must cite the notes they use.
- Agents must distinguish source-supported facts from recommendations.
- Agents must not silently write permanent notes without validation.
