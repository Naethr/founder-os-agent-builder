# Admin Rules

## Purpose

Set operating rules for quotes, scope, evidence, and sensitive actions.

## Scope Rules

- Document the requested pages, content needs, forms, integrations, review cycles, and maintenance expectations before quoting.
- Keep the first delivery small enough to complete and review.
- Separate setup work from monthly maintenance work.
- Treat unvalidated pricing as a planning assumption, not a commitment.

## Evidence Rules

- Store meaningful run evidence in `evidence/runs/`.
- Evidence should state the input, sources used, output obtained, limitations, cost, and data exposure.
- Do not create fake screenshots, fake tool runs, or fake cloud usage.

## Quote Rules

- A quote is not final until reviewed by the founder.
- Timelines require content readiness and scope clarity.
- Any financial commitment needs human validation.
